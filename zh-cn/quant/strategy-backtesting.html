Unit 4.3

TradingView 策略歷史回測与 Webhook 自動化跟单

深入学習量化交易的核心環节：如何科学解读策略回測報告，以及如何利用 Webhook 技术将 TradingView 讯号对接至盈透证券等券商，实現無人值守的自動化程式交易。

1. 量化交易的核心：歷史回測 (Backtesting)

在量化交易中，我们絕对不憑感覺下单。任何交易邏輯在投入实战资金前，都必須通过**歷史數據回測**的檢驗。

當我们在 TradingView 載入一個 Pine Script 策略後，下方的 **「策略測試器 (Strategy Tester)」** 會瞬间模擬过去幾年、甚至十幾年該策略的每次进出场，並跑出多項關鍵績效指标：

  • 淨利潤 (Net Profit)：策略在整個回測期间賺取或虧损的總金額。
  • 勝率 (Win Rate)：獲利交易筆數占總交易筆數的百分比。
  • 最大回撤 (Max Drawdown)：帳戶淨值從歷史最高点回落到最低点的最大百分比，這是评估帳戶是否會「斷头爆倉」的關鍵风控指标。
  • 獲利因子 (Profit Factor)：總獲利金額 / 總虧损金額。大於 1.5 代表這是一個相对穩健的策略。

2. 如何防範「过度擬合 (Overfitting)」？

許多初学者在寫 Pine Script 策略时，會拼命優化參數，讓歷史回測圖表看起來非常完美，這在量化領域稱為「过度擬合（过度優化）」。

⚠️ **警告**：过度擬合的策略在歷史數據中賺大錢，但在未來的真实市场中，往往會遭遇嚴重的虧损。

健全策略的三大黄金法则：

  1. 少即是多：限制您的过濾條件。條件越簡单、參數越少，策略的通用性越強。
  2. 樣本外測試 (Out-of-Sample Testing)：使用 2018-2023 年的數據开發策略，然後拿 2024-2026 年「完全未參与優化」的數據进行驗证。
  3. 多标的測試：将您的策略套用在台指期、小道瓊、黄金期货等不同市场，驗证其是否具備跨市场的生命力。

3. 自動化程式交易：Webhook 跟单運作原理

當您在 TradingView 驗证好一套具有優勢的策略後，不需要人工手動下单。我们可以透过 **Webhook 警示** 实現自動跟单：

自動化交易路徑：
1. TradingView 策略觸發买入讯号 📈
2. TradingView 伺服器發送 JSON 警示通知到指定的 **Webhook URL** 🌐
3. 接收端伺服器（例如 Python 雲端主機）收到通知，向您的券商 API 下達交易指令 📲
4. 盈透证券 (Interactive Brokers) 帳戶自動執行买入下单 🏛️

4. 如何设定 Webhook 自動下单警示？

  1. 打开您的 TradingView 策略圖表。
  2. 点擊圖表右上方的 **「警示 (Alert)」** 圖示。
  3. 在條件中选择您的 **「自訂策略 (Strategy)」**，此时會自動綁定策略的买賣买賣讯号。
  4. 在「通知」页籤中，勾选 **「Webhook URL」**，並輸入您的交易機器人网址（例如 AutoView, TradingView-to-Broker API 連接器等）。
  5. 在讯息框中，使用 JSON 變數代碼设定下单规格（例如合约口數、委託价格），点擊建立。當讯号來臨时，下单将在 1 秒內自動完成！

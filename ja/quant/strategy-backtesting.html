Unit 4.3

TradingView 策略歷史回測與 Webhook 自動化跟單

深入學習量化交易的核心環節：如何科學解讀策略回測報告，以及如何利用 Webhook 技術將 TradingView 訊號對接至盈透證券等券商，實現無人值守的自動化程式交易。

1. 量化交易的核心：歷史回測 (Backtesting)

在量化交易中，我們絕對不憑感覺下單。任何交易邏輯在投入實戰資金前，都必須通過**歷史數據回測**的檢驗。

當我們在 TradingView 載入一個 Pine Script 策略後，下方的 **「策略測試器 (Strategy Tester)」** 會瞬間模擬過去幾年、甚至十幾年該策略的每次進出場，並跑出多項關鍵績效指標：

  • 淨利潤 (Net Profit)：策略在整個回測期間賺取或虧損的總金額。
  • 勝率 (Win Rate)：獲利交易筆數占總交易筆數的百分比。
  • 最大回撤 (Max Drawdown)：帳戶淨值從歷史最高點回落到最低點的最大百分比，這是評估帳戶是否會「斷頭爆倉」的關鍵風控指標。
  • 獲利因子 (Profit Factor)：總獲利金額 / 總虧損金額。大於 1.5 代表這是一個相對穩健的策略。

2. 如何防範「過度擬合 (Overfitting)」？

許多初學者在寫 Pine Script 策略時，會拼命優化參數，讓歷史回測圖表看起來非常完美，這在量化領域稱為「過度擬合（過度優化）」。

⚠️ **警告**：過度擬合的策略在歷史數據中賺大錢，但在未來的真實市場中，往往會遭遇嚴重的虧損。

健全策略的三大黃金法則：

  1. 少即是多：限制您的過濾條件。條件越簡單、參數越少，策略的通用性越強。
  2. 樣本外測試 (Out-of-Sample Testing)：使用 2018-2023 年的數據開發策略，然後拿 2024-2026 年「完全未參與優化」的數據進行驗證。
  3. 多標的測試：將您的策略套用在台指期、小道瓊、黃金期貨等不同市場，驗證其是否具備跨市場的生命力。

3. 自動化程式交易：Webhook 跟單運作原理

當您在 TradingView 驗證好一套具有優勢的策略後，不需要人工手動下單。我們可以透過 **Webhook 警示** 實現自動跟單：

自動化交易路徑：
1. TradingView 策略觸發買入訊號 📈
2. TradingView 伺服器發送 JSON 警示通知到指定的 **Webhook URL** 🌐
3. 接收端伺服器（例如 Python 雲端主機）收到通知，向您的券商 API 下達交易指令 📲
4. 盈透證券 (Interactive Brokers) 帳戶自動執行買入下單 🏛️

4. 如何設定 Webhook 自動下單警示？

  1. 打開您的 TradingView 策略圖表。
  2. 點擊圖表右上方的 **「警示 (Alert)」** 圖示。
  3. 在條件中選擇您的 **「自訂策略 (Strategy)」**，此時會自動綁定策略的買賣買賣訊號。
  4. 在「通知」頁籤中，勾選 **「Webhook URL」**，並輸入您的交易機器人網址（例如 AutoView, TradingView-to-Broker API 連接器等）。
  5. 在訊息框中，使用 JSON 變數代碼設定下單規格（例如合約口數、委託價格），點擊建立。當訊號來臨時，下單將在 1 秒內自動完成！
